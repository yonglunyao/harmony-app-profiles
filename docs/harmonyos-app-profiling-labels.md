# HarmonyOS 应用画像标签体系

> 版本：v1.1
> 更新日期：2025-03-17
> 目标：构建100+核心标签的多维度应用画像

---

## 标签体系设计原则

1. **全面覆盖**：覆盖静态、动态、行为、舆情、开发者等多维度
2. **层次结构**：一级维度 → 二级类别 → 三级标签
3. **可检测性**：每个标签都有明确的检测方法和数据来源
4. **可扩展性**：支持新增标签和标签体系演进
5. **场景适配**：适配检测决策、安全分析、开发者反馈、用户提示等场景

---

## 标签数据结构

```json
{
  "label_id": "string",           // 标签唯一标识
  "label_name": "string",         // 标签名称
  "label_category": "string",     // 所属一级维度
  "label_subcategory": "string",  // 所属二级类别
  "description": "string",        // 标签描述
  "risk_level": "enum",           // 风险等级：critical/high/medium/low/info
  "weight": "number",             // 标签权重（用于风险评分）
  "detection_method": "enum",     // 检测方法：static/dynamic/behavior/reputation/expert
  "require_review": "boolean",    // 是否需要专家审核
  "data_source": "array",         // 数据来源
  "threshold": "object",          // 检测阈值配置
  "created_at": "datetime",       // 创建时间
  "updated_at": "datetime"        // 更新时间
}
```
